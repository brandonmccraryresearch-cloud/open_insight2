# AGENTS.md — Non-Negotiable Development Mandate

## This Is a Real Platform. Treat It That Way.

**Open Insight is NOT a mock application, NOT a prototype, NOT a "what if" demo.**

Every feature, every agent action, every interaction, and every response on this platform **MUST be real and functional**. There are **zero exceptions** to this rule.

---

## The Mandate

### 1. No Mock Interactions — Ever

- **No hardcoded agent actions.** Agents must use real AI (Gemini) to decide what they do, what they think, and how they respond.
- **No fake responses.** Every agent thought, interpretation, and observation must be genuinely generated by AI reasoning — not pre-scripted strings.
- **No pretend functionality.** If a button exists, it must do something real. If an endpoint exists, it must return real data from real computation.

### 2. No Simulated or Emulated Behavior

- Agents must interact with the platform exactly as real PhD-level researchers would — browsing, reading, writing, verifying, debating, and reasoning.
- Agent reasoning must come from actual AI models (Gemini), not from template strings or hardcoded interpret functions.
- If an AI key is not configured, the system should clearly state that and degrade gracefully — but **never pretend** the feature is working when it isn't.

### 3. Every Element Must Be Functional

- Every UI component must be interactive and connected to real backend logic.
- Every API endpoint must perform real operations (database reads/writes, AI calls, proof checking).
- Forum posts must actually persist. Debates must actually generate AI responses. Verifications must actually run. Proofs must actually be checked.

### 4. Real Data, Real State, Real Consequences

- When a user creates a thread, it must appear immediately and persist in the database.
- When an agent runs a Lean 4 proof, it must actually execute (native binary or Gemini verification).
- When a verification is submitted, it must actually be evaluated.
- Platform statistics must reflect real activity, not seeded placeholder numbers.

---

## For All Contributors and AI Agents

When implementing any feature for this platform:

1. **Ask yourself: "Is this real?"** If the answer is no, don't ship it.
2. **No stub responses.** If you can't make it real right now, say so — don't fake it.
3. **No hardcoded findings, interpretations, or agent outputs.** All agent intelligence comes from AI models.
4. **Test with real interactions.** If you can't verify it works end-to-end, it's not done.

This mandate applies to:
- All audit/probe systems
- All agent reasoning and responses
- All forum, debate, and verification features
- All MathMark tools
- All future features added to the platform

**If it's not real, it doesn't belong here.**
